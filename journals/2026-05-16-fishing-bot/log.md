# Задача: бот для рыбалки через CV + A/D
Дата: 2026-05-16
Статус: рабочий без humanizer; humanizer для этой игры слишком тормозит

## Контекст
- Игра: HoYo-подобная (UID 12-знач), русский UI: "Выносливость рыбы", "Леска", "Тяни влево A / Тяни вправо D".
- Полоса сверху экрана. В ней зелёная (cyan) целевая зона, внутри которой узкий жёлтый вертикальный ползунок.
- Цель: удерживать ползунок в зелёной зоне нажатиями A/D.
- Скриншот: `img.png` в корне.

## Решения
- **Ввод:** pydirectinput (SendInput со скан-кодами). Античит игры — серверно-поведенческий, kernel-mode нет. Если игра ввод не примет — план Б: Pi Pico HID; план В: Flipper Zero с кастомной прошивкой и FAP.
- **CV:** mss для захвата региона + OpenCV для HSV-масок (зелёная зона и жёлтый ползунок).
- **Логика:** error = slider_x - zone_center_x; зона мёртвой полосы (deadband), чтобы не дрожало. err>0 → A (тянуть влево), err<0 → D.
- **Хоткеи:** F8 — старт/пауза, F9 — выход. На случай если бот зависнет давя клавишу.

## Регион полосы (пропорции от скрина 1531x861)
- X: 0.268–0.741 (от ширины экрана)
- Y: 0.029–0.075 (от высоты)
- На 1920x1080: ~ (515, 31, 908x50)

Дефолт в config.py, пользователь переопределит через калибровку.

## Запуск
```
pip install -r requirements.txt
python calibrate.py file img.png   # проверить детектор на скрине
python calibrate.py screen          # проверить на живом экране (открыть рыбалку в игре)
python bot.py                       # F8=start/pause, F9=quit
```

## Если что-то не так
- Зона/ползунок не находятся → `calibrate.py screen`, посмотреть debug_overlay.png. Подкрутить HSV в config.py или Region.
- Игра не реагирует на клавиши → проблема pydirectinput. План Б: Pi Pico с CircuitPython HID. План В: Flipper Zero с FAP "USB Serial → HID".
- Нужны права админа для `keyboard` хоткеев на некоторых системах. Если F8/F9 не ловятся — запустить терминал от админа.

## Решения и находки
- На скриншоте 1632x911 полоса по факту: X 497–1142, Y 27–72. Соответствует пропорциям, заложенным в config.
- HSV зоны: H 70–96, S 80–209, V 151–255 (циан). HSV ползунка: H ~25, жёлтый.
- Иконки на концах полосы маскируют тот же цвет: левая (рыба) — жёлтая, правая (леска) — циан. Поэтому X-кроп нужно затянуть внутрь иконок, иначе ползунок «съезжает» влево в иконку рыбы.

## Humanizer v2 (умный)
Переписан целиком. Интерфейс: `step(slider_x, zone_center, deadband)` вместо `step(desired)` — теперь сам считает антиципацию.

Что моделирует:
- **Антиципация по скорости**: тек.поз + v × motor_lag → предсказанная позиция. Решение по предсказанной ошибке, не по текущей.
- **Шум предсказания** ±2.5 px σ.
- **Log-normal RT** (median 200, σ 0.28 в log-space). У реальных людей RT правоскошен, не uniform.
- **Warmup**: первые 2 сек сессии +80 мс к медиане.
- **Fatigue**: +1.2 мс/мин, потолок +60 мс.
- **Реверс-штраф**: смена направления +45 мс.
- **Rhythm-бонус**: продолжение того же направления после короткой паузы −25 мс.
- **Восстановление после промаха**: +60 мс на следующие 500 мс.
- **Длительность нажатия ∝ |ошибка|**: base 60 мс + 3.5 мс на каждый px ошибки, log-normal σ 0.22.
- **1.8% промахи** запускают recovery; **0.6% микропаузы** вместо нажатия.

Поведенческий тест:
- Сценарий «дрейф 30 px/s»: первая реакция на 373 мс (warmup активен).
- Сценарий «разворот в 0.6 с»: RT 366 мс, релиз в deadband — реверс работает.
- 7 случайных скачков цели: median RT 209 мс, p90 242, σ 26 мс.

Тюнинг через `CONFIG.humanizer.*`. Главные ручки:
- `reaction_median_ms` — общая скорость реакции.
- `anticipation_ms` — насколько «опережать» (если бот мажет — снижать; если запаздывает — поднимать).
- `press_per_px_ms` — крутость зависимости нажатия от ошибки.
- `miss_chance` — частота «человеческих» ошибок.

## Финал: humanizer выключен

После тестов на живой рыбалке: с humanizer'ом любого уровня бот **слишком медленный**. Маркер (жёлтый, "рыба") в этой игре прыгает на ~1000-1500 px/s, любая RT > 30 мс ломает реакцию. По умолчанию `humanizer.enabled = False`.

**Понимание игры (для будущих сессий):**
- Циан-полоса = СЛАЙДЕР, управляется A/D.
- Жёлтый внутри неё = МАРКЕР, движется сам (рыба сопротивляется).
- Цель: удерживать жёлтый внутри циан.
- A в этой игре двигает циан **вправо**, D — **влево** (подтверждено из лога).
- Логика бота при этом совпала: `err > 0` (маркер правее центра циан) → press A → циан едет вправо к маркеру. ✓

**Антидетект-стратегия без humanizer'а:**
- Поведенческий бан тут — главный риск (клиент pydirectinput не палит).
- Делать паузы между рыбами 5-15 сек.
- Не фармить 24/7.
- Если потом захочется humanizer — собрать только из безопасных в этой механике фич: `use_pause` (микропаузы), `use_press_duration_scaling`. **Без** reaction delay и `use_miss`.

Статус: завершено, в продакшене.

## Апдейт 2026-05-16: включены все флаги humanizer'а
По запросу пользователя включены оставшиеся флаги в `HumanizerConfig`:
`use_rt_jitter`, `use_anticipation`, `use_prediction_noise`, `use_warmup`,
`use_fatigue`, `use_reversal_penalty`, `use_rhythm_bonus`, `use_miss`.

⚠ Конфликт с предыдущим выводом: маркер 1000-1500 px/s, `rt_min_ms=70` →
RT-jitter добавит ≥70 мс на каждое решение, что по прошлым тестам ломало
реакцию. Если бот начнёт мазать — откатить `use_rt_jitter` и `use_miss`
в первую очередь, либо снизить `rt_min_ms`.

## Апдейт 2026-05-16: автозакрытие экрана награды через ESC
`RewardScreenWatcher` в `bot.py`. Триггер — тёмная капсула «Уровень
рыбалки X · NNN/MMMM · +K» вверху по центру (а не розовый XP-бар внутри
неё, т.к. при level-up он пустой). Регион X 0.40-0.66, Y 0.10-0.16,
HSV тёмных пикселей (V<70, S<100), минимум 2000 пикс. State machine:
idle → scheduled (плашка появилась → ESC через random(0.3, 1.5)) →
cooldown (ESC нажат, ждём пока плашка исчезнет) → idle.

Параметры в `CONFIG.reward_screen`: `region`, `panel_hsv`, `min_pixels`,
`esc_min_s`, `esc_max_s`, `esc_key`. Если детектится ложно — поднять
`min_pixels` или сузить регион/HSV.

## Апдейт 2026-05-16: автотап F → заменён на детектор иконки
Изначально был `AutoTapper` (F каждые 2-6 сек вслепую). Заменён на
`InteractPromptWatcher` — ловит иконку «нажми F» в правом нижнем углу
(регион X 0.85-1.0, Y 0.80-1.0) и жмёт F с задержкой 0.15-0.6 сек.
State machine как у RewardScreenWatcher: idle → scheduled → cooldown.

Параметры в `CONFIG.interact_prompt`: `region`, `icon_hsv`, `min_pixels`,
`delay_min_s`, `delay_max_s`, `key`, `hold_min_ms`, `hold_max_ms`.

⚠ HSV иконки задан по умолчанию (белый-светлый: H 0-180, S 0-60, V 220-255) —
скорее всего нужна калибровка под реальный цвет иконки промпта.

Калибровка после получения скриншота: иконка = синий круг с белым крючком
+ ярко-cyan кольцо прогресс-бара, кнопка F под ним. Регион сужен до
X 0.90-1.0, Y 0.83-1.0. HSV переведён на синий: H 95-120, S 150-255,
V 150-255, min_pixels=150. Признак — яркий синий кант кольца.

Второй кейс: ряд из 4 иконок R/Q/E/F (idle, можно начать рыбалку) —
светлые без cyan-кольца. Детектор переделан на OR двух HSV-масок:
`cyan_hsv` (H 95-120, S 150-255, V 150-255) + `white_hsv`
(H 0-180, S 0-50, V 210-255). Регион стянут плотнее: X 0.92-1.0,
Y 0.86-0.99, чтобы исключить микрофон/UI левее. min_pixels=120.

Третий кейс: тёмно-серая иконка крючка с тонким cyan-полу-arc сверху
(прогресс только начался). Белый крючок не такой яркий — V~180-220.
Параметры расслаблены: `white_hsv` V 180-255 (было 210), `cyan_hsv`
H 90-125 (было 95-120) и S от 120 (было 150). Регион ужат: X 0.93-0.99,
Y 0.86-0.96 — исключает микрофон снизу и обрезку справа. min_pixels=70.

Дальнейшее ослабление + дневной фон (пользователь: «пофиг на ложные»):
- region X 0.90-1.0, Y 0.83-0.99
- white_hsv V 140-255, S 0-80
- cyan_hsv H 85-130, S 80-255, V 100-255
- min_pixels=25
- debug=True (логирует `cyan=N white=M or=K`)

⚠ Важно: COOLDOWN переделан с visible-based на time-based. Старый
вариант ждал «пока иконка пропадёт», но дневная голубая вода
постоянно матчится в cyan_hsv → watcher залипал. Теперь после тапа
жёсткий cooldown 1.5-3.0 сек по таймеру, после — снова IDLE.

## Апдейт 2026-05-16: маркер вне зоны — fallback на весь region
`BarDetector.detect` искал маркер только в окне `zone ± 8 px`. Если маркер
вылетал дальше — `slider_x=None`, бот переставал реагировать (визуально
«маркер замирает»). Добавлен fallback: если в окне маркер не найден —
ищется по всей маске region. Region уже сужен внутрь полосы (X 0.305-0.7),
поэтому иконки рыбы/лески по краям не попадают и не дают ложного маркера.

## Апдейт 2026-05-16: баннер «Рыба на крючке!»
Второй детектор F: `FishCaughtBannerWatcher`. Ловит тёмную плашку
вверху по центру с текстом «Рыба на крючке! Нажмите кнопку…».
Признак — большое количество тёмных пикселей (V<70, S<80) в регионе
X 0.28-0.72, Y 0.21-0.30. Параметры в `CONFIG.fish_caught_banner`.
Реакция быстрее, чем у обычного промпта (delay 0.10-0.50 сек), потому
что рыба может сорваться. Cooldown 3-6 сек.

Работает параллельно с `InteractPromptWatcher` — двойная страховка.
