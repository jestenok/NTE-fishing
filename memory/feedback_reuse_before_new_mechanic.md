type: feedback
---
Когда добавляешь поддержку новой игры в vision-bot: не создавай новый модуль
в mechanics/, если поведение совпадает с уже существующим (RegionWatcher,
SliderMechanic). Сначала проверь, нельзя ли обобщить существующий модуль.

Пример: «тест на реакцию» (регион позеленел → клик) сделал отдельной
mechanics/reaction.py — это оказалось копией автомата RegionWatcher.
Правильно: вынести действие в Action (core/actions.py: KeyPress / MouseClick)
и сделать reaction-test обычным WatcherConfig.

**Why:** Отдельная механика оправдана только при иной логике (как SliderMechanic
с непрерывным управлением и humanizer). RegionWatcher = «регион стал X → действие»
покрывает и UI-сторожей, и reaction-test. Дубликат конечного автомата — техдолг.

**How to apply:** Перед созданием файла в mechanics/ сравни автомат/поведение с
RegionWatcher. Если отличается только действие или пороги — обобщай существующий
модуль (новый Action, новое поле конфига), а не копируй.
